# bracket-counter-plugin

Тестовое задание МТС: Gradle-плагин на Kotlin DSL, который для
Java-модулей кладёт в итоговый JAR текстовый файл с количеством открывающих
скобок в каждом исходнике.

Выглядит это так (opening-brackets.txt внутри jar):

```
com/example/Hello.java: 3
com/example/util/Calc.java: 9
```

## Как устроено

Проект собран как composite build:

- `plugin/` - сам плагин (`includeBuild` в корневом settings), поэтому
  sample-модуль применяет его по id без публикации куда-либо
- `sample-lib/` - обычный java-library модуль для демонстрации

Плагин вешается на любой модуль, но что-то делает только если там есть
java-плагин (`plugins.withType(JavaPlugin)`). Тогда регистрируется задача
`countOpeningBrackets`, которая пишет отчёт в
`build/bracket-counter/opening-brackets.txt`, а jar-таска забирает его через
`from(countTask.flatMap { it.reportFile })`.

## Запуск

Нужен JDK 21. Gradle качается wrapper'ом.

```
./gradlew :sample-lib:jar
cat sample-lib/build/bracket-counter/opening-brackets.txt
```

Убедиться, что файл находится в джарнике:

```
unzip -p sample-lib/build/libs/sample-lib.jar opening-brackets.txt
```

Тесты плагина (Gradle TestKit, поднимают сборки):

```
./gradlew :plugin:test
```

## Настройка

По умолчанию считаются `(`, `{` и `[`. Можно поменять в build.gradle.kts
модуля:

```kotlin
bracketCounter {
    openingBrackets.set("{")
}
```

## Допущения

- В ТЗ не сказано, какие именно скобки считать, поэтому по умолчанию беру
  все три вида `(`, `{`, `[`. Если имелись в виду только фигурные,
  это меняется одной строчкой в конфиге (см. «Настройка»), само поведение
  плагина от трактовки не зависит.
- Подсчёт наивный, посимвольный: скобки в строковых литералах и комментариях
  тоже попадают в счёт. Задача сформулирована как «количество скобок в файле»,
  а не «в коде», так что парсер Java тут был бы оверинжинирингом.
- Считаются только исходники из main source set (`src/main/java`). Тесты
  и сгенерированный код не учитываются.
- Пути в отчёте - относительные к корню исходников и всегда с прямыми
  слэшами, строки отсортированы. Так отчёт получается одинаковым на любой ОС
  и при повторных сборках.
